{{ icon('globe') }} Connect to WordPress

Fetch posts directly via the WP REST API.

Recommended path. Uses your WordPress Application Password (Users → Profile → Application Passwords) for authentication, which means you don't have to expose your real admin password.

Drafts and private posts only come back when authenticated. The fetch is capped at 500 posts; rerun the wizard or filter on the WordPress side if you need more.

{{ icon('cloud-upload') }} Upload a CSV

Use a WP All Export "Posts" file or a generic posts CSV.

Best when the WP REST API is locked down or the site is offline. Accepts either WP All Export's column names (Title, Categories, Content, Date, Featured Image, Author, …) or the raw WordPress columns (post_title, post_content, post_category, …).

Featured-image URLs in the CSV are downloaded only at commit time, not when the CSV is uploaded.

{{ icon('help-circle') }} How this works

  1. 1 Choose a source. Connect to a live WP site or upload a CSV.
  2. 2 Review & map. Each post can be sent to Stories, Announcements, Events, or skipped. Bulk-map an entire WordPress category in one click.
  3. 3 Dry run. The wizard reports exactly what it would create — title, target, slug, image — so you can correct course before any rows land.
  4. 4 Import. Type IMPORT to confirm. Featured images are downloaded and deduplicated by content hash; duplicate slugs auto-suffix with -2, -3, etc.
