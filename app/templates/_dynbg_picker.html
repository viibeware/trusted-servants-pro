{# Dynamic-background trigger button. Drop-in replacement for any admin form that needs a "pick a dynamic background" affordance. Renders: 1. Five hidden s carrying the current selection — base key, overlay key, and three custom colour slots — so a normal form POST submits everything in one shot. The consumer routes each value through ``app.dynbg.normalize*`` / ``normalize_color`` server-side to clamp tampered values. 2. A button that opens the global dynbg modal (rendered once in base.html). The modal reads the trigger's current values, lets the admin pick a different background / overlay / paint a custom palette, and writes the chosen values back to the hidden inputs on Save. The trigger preview re-uses the same inline thumbnail recipe the modal does, so the button shows what's currently selected without having to crack open the modal. Usage: {% from "_dynbg_picker.html" import dynbg_trigger with context %} {{ dynbg_trigger(field_name='bg_dynamic_key', current=(page.bg_dynamic_key if page else None), config_field='bg_dynbg_config_json', config=(page.bg_dynbg_config_json if page else None)) }} Optional kwargs: button_id — DOM id for the trigger; auto-generated when blank. Pass an explicit id when multiple triggers live on the same page so the JS pairing stays unambiguous. label — text shown next to the preview (default: 'Choose…'). When a preset is already selected the button text swaps to its display name. #} {%- macro dynbg_trigger(field_name='bg_dynamic_key', current=None, config_field='bg_dynbg_config_json', config=None, button_id=None, label='Choose…') -%} {%- set _bid = button_id or ('dynbg-trigger-' ~ field_name|replace('[','-')|replace(']','-')|replace('.','-')) -%} {# Resolve the catalog entry for the currently-saved key. Jinja's `{% set %}` inside `{% for %}` doesn't leak outside the loop scope (loops are their own scope unless namespace() is used), so we resolve via selectattr|first which DOES return a value the surrounding template can read. Without this fix, `_entry` was always None even when `current` matched a catalog key, which caused the trigger to render "Choose…" / "No dynamic background" after every save instead of reflecting the persisted value. #} {%- set _entry = (dynbg_catalog() | selectattr('key', 'equalto', current) | list | first) if current else none -%} {%- set _cfg = dynbg_decode(config) -%} {%- set _overlay = _cfg.overlay or '' -%} {%- set _overlay_entry = (dynbg_overlays() | selectattr('key', 'equalto', _overlay) | list | first) if _overlay else none -%} {%- set _colors = _cfg.colors or [] -%} {%- set _c1 = _colors[0] if _colors|length > 0 else '' -%} {%- set _c2 = _colors[1] if _colors|length > 1 else '' -%} {%- set _c3 = _colors[2] if _colors|length > 2 else '' -%} {%- set _scope = _cfg.overlay_scope or '' -%} {%- set _nsize = _cfg.overlay_size if _cfg.overlay_size is not none else '' -%} {%- set _nint = _cfg.overlay_intensity if _cfg.overlay_intensity is not none else '' -%} {%- set _rand_c = '1' if _cfg.randomize_colors else '' -%} {%- set _rand_p = '1' if _cfg.randomize_positions else '' -%} {# Animations default to ON. Persist '0' (disabled) as the only non-default state; '' = default = animated. #} {%- set _anim = '0' if _cfg.animate is sameas false else '' -%} {# Pastel-strength slider value (0-100). decode_config returns an int; a legacy True boolean previously stored was normalised to 100 by decode_config's `normalize_pastel_strength`. Empty string when the strength is 0 so the trigger button's status text can detect "off" without parsing. #} {%- set _pastel_int = _cfg.pastel_light | int -%} {%- set _pastel = (_pastel_int | string) if _pastel_int else '' -%} {# Per-preset knob values (dot size/gap, line angle/thickness, …) travel as a single JSON blob so the macro + save handler don't need a hidden input per knob. Empty object → '' so the status text / server treat it as "no overrides". #} {%- set _knobs = _cfg.knobs or {} -%} {%- set _knobs_json = (_knobs | tojson) if _knobs else '' -%}
