Roles & permissions

What each role can access. Module access is also gated per-module on the Modules tab.
  • Admin
    Full access. Manages users, modules, settings, the Web Frontend, and every other feature in the portal.
  • Editor
    Edits meetings, libraries, readings, and uploads. Cannot reach Settings, Users, or the Web Frontend module. Library-file deletion: may delete only readings whose creator was an Editor / Frontend Editor / Intergroup Member — admin-created and legacy files are protected.
  • Frontend editor
    Inherits Editor permissions plus the entire Web Frontend module — header, footer, navigation, mega menus, alert bars, homepage content, theme/design tokens, and the public-visibility toggle. Cannot delete library files regardless of who uploaded them.
  • Intergroup member
    Inherits every Editor permission (meetings, libraries, readings, uploads), plus exclusive edit access to the Intergroup Documents and Intergroup Minutes libraries — which regular Editors and Frontend Editors cannot touch. May delete any library file. Cannot reach Settings, Users, the Web Frontend module, or the Intergroup Email Accounts page (admin-only).
  • Viewer
    Read-only. Sees meetings, libraries, files, and any module the admin opens to All signed-in users; cannot edit anything.
