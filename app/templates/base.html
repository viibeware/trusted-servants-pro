    Changelog

    • 1.3.11
      April 2026
      • Mobile sidebar footer always visible — on screens ≤ 900px the sidebar is now a three-zone flex column: the brand header and the user-chip / help / theme / sign-out footer are pinned, and only the nav scrolls between them. Sidebar height switched to 100dvh so it honors the actual visible viewport when the browser address bar collapses in and out.
      • Create User from an access request — each row on the Access Requests page gains a Create User action. Clicking it opens Settings → Users with the iframe reloaded against ?prefill=<email>, pre-populating both the username and email fields and focusing the password input so the admin can just type a password and submit.
      • OTP Email widget on Zoom Accounts — the accounts card now has a two-column layout: the accounts table on the left, a boxed OTP Email widget on the right with the copyable email and password, password reveal, and a "Login for OTP Codes" button — same look and behavior as the meeting detail page. Collapses to a single column at ≤ 900px.
      • Notes as a tooltip icon — in the Zoom Accounts table, the Notes column is now an info (ⓘ) icon button that only appears when a note is set. Clicking the icon toggles a tooltip with the note text (multi-line preserved); clicking outside or pressing Escape closes it. Reuses the existing heading-help tooltip infrastructure.
      • Settings modal mobile fixes — the tab strip can no longer scroll vertically (it now has an explicit overflow-y: hidden) so stray touch drags stay on the horizontal axis. The Appearance pane no longer overflows horizontally: the pane gets overflow-x: hidden, its 2-column grid uses minmax(0, 1fr) tracks, and all grid / pane children get min-width: 0 so long inline content can't blow the container width.
    • 1.3.10
      April 2026
      • Turnstile fully refreshes after failed login — the previous fix tried to re-render the widget in-place after a document.write(), but the Cloudflare API's internal widget tracking didn't cooperate and the old single-use token stayed on screen. The login page now extracts the server's flash message from the failed POST response, stashes it in sessionStorage, and triggers a full page reload. The fresh page gets a brand-new Turnstile widget, a fresh CSRF token, and the stashed flash is rendered into the flash strip on page load.
    • 1.3.9
      April 2026
      • Case-insensitive username login — usernames are now matched case-insensitively at login via func.lower(User.username). Existing accounts with mixed-case usernames keep their display case but users can sign in with any casing. User creation and email updates also check for case-insensitive collisions so "Admin" and "admin" can no longer be created as two separate accounts.
      • Turnstile re-renders after failed login — the fetch-driven login flow does document.write() on the server's re-rendered response, which the Cloudflare Turnstile API's singleton script doesn't automatically pick up. The page now explicitly calls turnstile.render() on every .cf-turnstile widget after the document is rewritten, issuing a fresh single-use challenge token without requiring a manual page reload.
    • 1.3.8
      April 2026
      • Zoom password reveal opened up — the /zoom-accounts/<id>/reveal endpoint is now gated with @login_required instead of @admin_required, so viewers and editors can use the "Show" button on the Zoom Accounts page. This matches the existing behavior on meeting detail pages (which already render the decrypted Zoom password for all authenticated users) and the OTP email reveal endpoint.
    • 1.3.7
      April 2026
      • Security hardening — portal-wide hardening pass.
        • CSRF protection (Flask-WTF CSRFProtect) is now active on every POST form and on AJAX fetches via a global X-CSRFToken header auto-injector. Every form template carries a hidden csrf_token input; the base template exposes the token through a <meta name="csrf-token"> tag.
        • Security headers on every response — Content-Security-Policy, X-Content-Type-Options: nosniff, X-Frame-Options: SAMEORIGIN, Referrer-Policy: strict-origin-when-cross-origin, Permissions-Policy, and Strict-Transport-Security (only on TLS connections).
        • Secure cookies — SESSION_COOKIE_SECURE, REMEMBER_COOKIE_SECURE, and HTTPONLY are now set by default (disabled only when TSP_DEBUG=1).
        • SECRET_KEY enforced — the app refuses to boot if TSP_SECRET_KEY is unset or still the placeholder value, except in debug mode. No more silent "dev-secret-change-me" in production.
        • Login brute-force protection — in-memory rate limiter caps to 10 failed attempts per IP per 15-minute window, returning HTTP 429. Successful logins clear the counter.
        • Open-redirect fix — the ?next= parameter on /auth/login is validated to be a same-host path. Off-host targets are ignored and the user is redirected to the dashboard.
        • XSS hardening — the legacy zoom_tech_content raw-HTML fallback no longer uses |safe; it now passes through a new |safe_rich_html bleach filter. Markdown output is also bleached against the same allowlist, so editor-authored reading bodies can't inject scripts.
        • Upload extension denylist — _save_upload rejects scripts, executables, and raw HTML/SVG/XML uploads (.exe, .sh, .php, .py, .html, .svg, etc.) with HTTP 400.
        • postMessage origin pinning — iframe ↔ parent messages (media picker, nav-links live update) are now scoped to window.location.origin instead of "*", and receivers validate event.origin.
        • Debug default — run.py now opts into debug mode only when TSP_DEBUG=1, not unconditionally.
    • 1.3.6
      April 2026
      • Online Now tile in Server Stats — the Server Stats card gains a fifth tile (admin-only) showing the number of users active in the last five minutes, with the list of usernames and roles exposed on hover. Refreshes every 30 seconds via /api/online-users. Activity is tracked via a new User.last_seen_at column updated by a throttled (once/minute per user) before_app_request hook, so it survives gunicorn restarts and works across workers.
      • Server Stats restricted to editors and admins — the CPU / memory / load / uptime column in the "Your Role and Server Stats" card is now hidden from viewers, and /api/server-metrics is gated with @editor_required. Viewers still see their role and capabilities in the same widget (now rendered full-width without the server column), and can still toggle the widget from Customize Dashboard; the toggle's label and description adapt to the viewer's role so it only mentions "Server Stats" for users who can see them.
    • 1.3.5
      April 2026
      • Open Graph / link previews — new admin-only section in Settings → Appearance controls the title, description, and image other platforms show when a portal URL is pasted into Slack, iMessage, Facebook, Twitter/X, etc. Emits og:* and twitter:* meta tags site-wide when enabled. The image is served at /site-branding/og-image (public, so crawlers can fetch it without auth). Four new SiteSetting columns (og_enabled, og_title, og_description, og_image_filename) with matching migration entries.
    • 1.3.4
      April 2026
      • First-run setup wizard — the first time an admin logs in with setup_complete = False, they are routed through a six-step focused wizard: set a strong admin password (required, 12+ chars with a live strength meter), Public Information Chair, SMTP, theme, sidebar branding, and Cloudflare Turnstile. Steps 2–6 are skippable; step 1 is gated and validates that the new password differs from the seeded default and isn’t a common term. Finishing flips SiteSetting.setup_complete so the wizard never re-fires.
      • Guided tour for viewers and editors — on first authenticated visit to the dashboard, non-admins get a nine-step spotlight-style walkthrough of the sidebar, dashboard, meetings, libraries, File Browser, Zoom Accounts, Settings, and the Help button. Completion is tracked in a tsp-tour-seen cookie (5-year expiry). The Help (?) button in the sidebar footer now opens a modal with a Take the tour button so anyone can replay it at any time.
      • File Browser lightbox — images (.jpg/.jpeg/.png/.gif/.webp/.svg/.bmp/.avif) and PDFs now open in a full-viewport modal with the filename, an “Open in new tab” link, and a Download button. Other formats keep their existing target="_blank" behavior. Cmd/Ctrl/Shift/middle-click still opens in a new tab. Scoped to the File Browser page only.
      • File Browser moved to /files — the central media URL is now /files with matching sub-routes (/files/upload, /files/<id>/rename, /files/<id>/delete, /files/<id>/download). Public file URLs at /pub/<filename> are unchanged.
      • Uninstaller is aggressive by default — uninstall.sh now purges Docker images and reverts the 80/443 UFW rules out of the box, and sweeps every Caddy-named volume so all Let’s Encrypt certificates, ACME account data, and Caddy config are wiped. A subsequent install.sh run starts from a clean slate and re-issues a fresh certificate. New opt-outs: --keep-images, --keep-ufw; --remove-docker remains opt-in.
    • 1.3.3
      April 2026
      • Zoom Accounts promoted to a sidebar entry — the page is no longer tucked inside the Settings modal. It now has its own top-level nav link visible to every logged-in user. Admins retain exclusive ability to create/edit/delete accounts; editors and viewers see the accounts table and the weekly calendar read-only.
      • New Security tab in Settings — a dedicated admin-only tab (sitting just before Data) houses Zoom OTP Email (moved off the Zoom Accounts page) and Login Bot Protection / Turnstile (moved out of Appearance). The two settings are laid out as a side-by-side grid on desktop with a vertical divider, collapsing to a stacked single column on narrow viewports.
      • Calendar slot time range — each cell in the Zoom Accounts weekly calendar now shows the full start–end window (e.g. 18:30–20:00) instead of start + duration. When a schedule has a per-day “Zoom opens” time set, the block’s start reflects that earlier opening time so hosts can see the real occupancy window at a glance.
      • AGPLv3 credit on About — the About pane now links to the canonical GNU AGPLv3 text under the Built-by credit.
    • 1.3.2
      April 2026
      • Per-user dashboard customization — each account now stores its own widget visibility and drag-and-drop order. Previously dashboard preferences were stored on SiteSetting and affected every user. The Customize button and the reorder handles are now available to all roles (admin, editor, viewer), and every user can tailor their home view independently.
      • Server Metrics widget opened up — the CPU / memory / load / uptime card on the dashboard is no longer admin-only. Viewers and editors can now enable it from Customize, and /api/server-metrics accepts any authenticated user.
      • Centralized version string — the app version now lives in a single app/version.py file and is injected into templates as {{ app_version }}. Sidebar, login footer, and About pane all read from that one source, so future version bumps require editing one line.
      • Worker-safe schema migrations — _migrate_sqlite() now catches the “duplicate column” error that can arise when two gunicorn workers race on the same ALTER TABLE during first boot after a schema bump. The loser of the race no longer crashes the worker.
      • Access logs — gunicorn now writes access logs to stdout so docker logs tspro shows incoming requests, which makes diagnosing client/server issues much faster.
      • Intergroup widget removed from the dashboard — the “Intergroup” quick-link card no longer appears as a dashboard widget. The dedicated /intergroup page and its sidebar entry are unchanged.
      • Installer DNS pre-flight — install.sh now resolves the requested TSP_DOMAIN and compares it to the server’s public IP before handing off to Caddy. If the record is missing or points at a Cloudflare proxy IP, it falls back to a self-signed certificate automatically so the portal stays reachable, and the final install banner explains how to recover (including the Cloudflare orange-cloud vs grey-cloud gotcha). The README’s DNS section was rewritten to match.
      • Request Access modal scrolls on short viewports — the modal is now capped at 90vh with the form body as the scroll region, so on laptop screens the submit button no longer gets clipped. The scrollbar track is hidden when content fits.
      • Login animation fires only on success with Turnstile enabled — previously invalid credentials still played the door-opening transition because the Turnstile path submitted natively and animated optimistically. The animation now waits for server confirmation whether or not Turnstile is active; on failure the response is rendered in place with the flash error and a fresh Turnstile token.
    • 1.3.1
      April 2026

Single-process dev runs were unaffected.
      • Deploy compose pinned to :latest — docker-compose.deploy.yml now tracks the latest tag so upgrades pick up automatically without a manual version bump.
