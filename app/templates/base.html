    Version 1.2

    Changelog

    • 1.2
      April 2026
      • Unbundled third-party branding — removed the DCCMA logo assets from the shipping image. Only the Trusted Servants and Viibeware marks are bundled now.
      • Sidebar logo on login — the admin-configured sidebar footer logo (or the Viibeware default) now appears at the bottom of the login hero at 100 px wide. SVGs are auto-whitened with a CSS filter so they read cleanly over the gradient.
    • 1.1
      April 2026
      • Login bot protection — optional Cloudflare Turnstile challenge on the login screen. Admin-configurable site key + Fernet-encrypted secret key, with a guard that refuses to enable the widget unless both are set.
      • Cache hardening — dynamic responses now send Cache-Control: no-store so authenticated pages aren’t retained by browsers or intermediaries after logout.
      • Unattended installer — install.sh for Ubuntu 24.04 sets up Docker, pulls the published image, and brings the portal up end-to-end without interactive prompts.
    • 1.0 — First public release
      April 2026
      • Meetings — full CRUD with logos, alert banners, per-day schedules (including opens time), in-person / online / hybrid types, archive & restore, table and card views, sort by name / day / type, and attached Zoom account + location selection.
      • Libraries & Readings — grouped file collections with drag-and-drop reorder, inline edit, granular reading selection per meeting, thumbnails, inline body text, and external link support.
      • File Browser — central media library with search, sort, grid & table views, rename, upload progress, and public direct-link URLs served under /pub/<filename> so files are shareable without authentication.
      • Copy Link buttons — every file exposes an inline clipboard button (Meetings, Libraries, File Browser) with clean human-readable URLs.
      • Roles & Access — admin / editor / viewer. Server-side decorators enforce permissions; UI hides disallowed controls.
      • Login screen — redesigned split layout with branded hero pane and credential pane. Configurable particle effects (Network, Starfield, Fireflies, Bubbles, Snow, Waves, Orbits, Rain) with adjustable speed and size, mouse-reactive physics, and a hand-tuned sine-wave gradient background (default teal→blue→purple) with multi-color picker, random palette generator, and live preview in Settings.
      • 3D login transition — optional door-opening animation with moving rainbow sine-wave warp and centered branding logo, fading to the active theme background before navigating to the next page.
      • Request Access — public form on the login screen capturing name, contact, role(s), and meeting; submissions recorded and emailed to a configurable recipient list.
      • Access Requests admin page — sidebar entry with pending-count badge and a dashboard widget for quick triage (Mark Handled / Reopen / Delete).
      • Zoom accounts — encrypted credential storage (Fernet), weekly assignment calendar starting Sunday, automatic conflict detection with red highlighting across overlapping slots, password reveal, and separate OTP email credentials displayed on online/hybrid meeting pages.
      • Intergroup page — configurable email address list with webmail link and mail-client setup instructions.
      • Zoom Tech Training — rich-block page with Standard and Wiki layouts, import scripts for WordPress content.
      • Dashboard — customizable widgets: stats, recent meetings, libraries, recent files, intergroup, access requests, and public information chair contact, each toggleable per site.
      • Themes — six full palettes (Light, Dark, Neobrutal Light/Dark, Cyberpunk, Solarpunk) with live swatches and remembered preference. Theme carries through to the login screen and persists via localStorage.
      • Appearance tab — custom sidebar footer logo + link (with width slider & preview) and the login screen particle/color configurator, all admin-only.
      • Email / SMTP — global SMTP server configuration with STARTTLS / SSL / plain, encrypted password storage, From name + address, "send to" recipient list for access requests, and a Send-Test utility.
      • Data export / import — full-portal zip (SQLite DB + uploads + zoom.key) with a safe in-place import that backs up the previous data to a timestamped folder before swapping.
      • Session length — 6-month remember-me cookie so users aren't repeatedly prompted for credentials.
      • Responsive design — dedicated mobile layouts for every view (Meetings, Libraries, File Browser, Users, Zoom Accounts, Locations) with stacked card rows replacing horizontal tables, a collapsible sidebar with outside-click-to-close, a two-bar menu icon, full-viewport settings modal with horizontally-scrollable tabs and fade hint, and full-width primary actions on mobile.
      • Settings UX — AJAX form submits that never close the modal, an in-modal toast notification system with success / warning / error variants, and live validation feedback.
      • Typography & color system — Inter font across the app (weights 100–900), unified accent color (#0b5cff) with matching hover and soft tints, consistent card padding at 2rem on desktop.
